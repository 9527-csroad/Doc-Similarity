# 文档相似度系统 - 项目规则

## 项目概述

本项目是一个支持 10 万+ PDF 文档的相似度检索系统，提供 RESTful API 接口。

---

## 架构原则

### 1. 模块化设计
- 处理器（Processors）：PDF解析、OCR、向量化各自独立
- 服务层（Services）：Milvus、MinIO、Redis 封装为独立服务
- API 层：按功能划分路由模块

### 2. 可扩展性
- 向量模型通过抽象接口支持切换（BGE/OpenAI/智谱）
- 配置通过环境变量管理，支持热更新
- 异步任务队列处理耗时操作

### 3. 阈值优先级
```
请求参数 > 用户配置 > 全局配置
```

---

## 开发规范

### 代码风格
- 使用 Python 类型注解
- Pydantic 进行数据验证
- 异步优先（async/await）

### 文件命名
- 模块文件：小写下划线（pdf_processor.py）
- 类名：大驼峰（PDFProcessor）
- 配置键：大写下划线（EMBEDDING_PROVIDER）

### API 设计
- RESTful 风格
- 版本前缀：/api/v1/
- 统一错误响应格式

---

## 技术决策记录

### 为什么选择 Milvus？
- 10万+ 规模需要专业向量数据库
- 支持 COSINE 相似度
- 毫秒级检索响应

### 为什么选择 BGE-M3？
- 多语言支持（中英混合）
- 本地部署，无 API 成本
- 1024 维向量，平衡精度和性能

### 为什么选择 PaddleOCR？
- 中文识别准确率高
- 支持多语言
- 开源免费
