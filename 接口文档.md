# 接口文档（v2 书籍）

**用途**：对外 HTTP 契约（上传 / 相似查询 / 热门）。实现细节见 `项目结构说明.md`，本地见 `DEMO_本地运行指南.md`。

三个接口成功响应 JSON **字段顺序**固定为：先 `message`，再 `time`（本次请求服务端**处理耗时**，单位秒，单调时钟测量），其余字段保持下文所列相对顺序。

---

## 一、相似度与默认模式

- **默认**：`upload` / `search` / `hotlist` 的 `fingerprint_mode` 未传时均为 `**merged`**。
- **merged**：伪页采样片段（前 3 + 中 3 + 后 2 + 标题等，见实现）**拼接成一段文本后做一次 embedding**，向量写入 `fingerprint_merged` 索引。
- **pooled**：各伪页片段**分别 embedding 再按权重池化**为一条向量，写入 `fingerprint_pooled` 索引。
- **检索分数**：FAISS 返回的为向量**内积**；向量已归一化时等价**余弦相似度**，范围约 [0,1]，**1.0 表示完全同向（含查询书自身）**。
- **口径一致**：`search`、`hotlist`（`threshold<1.0` 分支）应用的索引须与你关心的数据一致；一般全链路用默认 `merged` 即可。

## 二、POST /api/v2/books/upload

**行为简述**：下载 `pdf_url` 算 SHA256、抽元数据后删临时文件；下载 `txt_url` 正文做指纹与落盘 `data/texts/{book_id}.txt`。**merged / pooled 两套向量都会写入**，与本次请求的 `fingerprint_mode` 无关；该字段只影响**去重时**与已有书比对所用向量（`content_hash` / `isbn` 命中时不走向量）。

**Request JSON**


| 字段               | 类型     | 必填  | 说明                                          |
| ---------------- | ------ | --- | ------------------------------------------- |
| pdf_id           | string | 是   | 业务侧唯一；库中已存在则 HTTP 200 且 `message` 为错误说明     |
| pdf_url          | string | 是   | 可下载 PDF；仅用于哈希与元数据                           |
| txt_url          | string | 是   | 正文 URL（建议 UTF-8）；指纹与落盘均基于此                  |
| fingerprint_mode | string | 否   | `merged` | `pooled`，默认 `merged`；去重向量匹配用对应索引 |


**Response JSON**


| 字段           | 说明                                                 |
| ------------ | -------------------------------------------------- |
| message      | 成功为 `success`；失败为具体错误文案                            |
| time         | 处理耗时（秒）                                            |
| book_id      | 归属书目 UUID；失败（如 pdf_id 重复）时为 `null`                 |
| pdf_id       | 回显请求中的 pdf_id                                      |
| is_duplicate | `true`：判为已有书，仅新增一条上传记录，**未再写指纹索引**                 |
| match_reason | 命中原因：`content_hash` / `isbn` / `vector`；新书为 `null` |
| upload_count | 该书目在库中**累计上传次数**（同一 book 多条 pdf_id 递增）             |


**示例（去重命中）**

```json
{
  "message": "success",
  "time": 1.234567,
  "book_id": "f68dedc0-f8f5-4df7-a401-c77653a9b450",
  "pdf_id": "3",
  "is_duplicate": true,
  "match_reason": "content_hash",
  "upload_count": 3
}
```

## 三、POST /api/v2/books/search

**行为简述**：根据 `pdf_id` 找到对应 `book_id`，取出该书在 `**fingerprint_mode` 对应索引**中的向量，FAISS 检索 top 500，保留分数 ≥ `threshold` 的书目；再**按书目展开为多条记录，每条对应一个 `BookUpload`（一个 pdf_id）**。查询书自身若未出现在匹配结果中会**强制加入且 similarity=1.0**。

**Request JSON**


| 字段               | 类型     | 必填  | 说明                                |
| ---------------- | ------ | --- | --------------------------------- |
| pdf_id           | string | 是   | 必须已上传过                            |
| threshold        | float  | 否   | 相似度下限，默认 **0.8**，范围 0,1           |
| fingerprint_mode | string | 否   | `merged` | `pooled`，默认 **merged** |


**Response JSON**


| 字段           | 说明                                               |
| ------------ | ------------------------------------------------ |
| message      | HTTP 200 时为 `success`；4xx/5xx 见 FastAPI `detail` |
| time         | 处理耗时（秒）                                          |
| query_pdf_id | 请求的 pdf_id                                       |
| threshold    | 使用的阈值                                            |
| total        | `results` 条数（多 pdf_id 指向同一 book 时会多条）            |
| results      | 有序列表：先按 similarity 降序，再按 upload_date             |


**results[] 每项**


| 字段                    | 说明                                          |
| --------------------- | ------------------------------------------- |
| pdf_id                | 该次上传的业务 ID                                  |
| pdf_url               | 该次上传的 PDF URL（来自上传入参，存于 `book_uploads.pdf_url`） |
| book_id               | 所属书目                                        |
| upload_date           | 该 pdf_id 上传时间                               |
| similarity            | 与查询书向量的内积得分（同 book 多为 1.0）                  |
| title / author / isbn | 书目元数据                                       |
| is_self               | **是否与查询 pdf 属于同一 book_id**（同一本书多次上传均为 true） |


**示例**

```json
{
  "message": "success",
  "time": 0.015,
  "query_pdf_id": "1",
  "threshold": 0.8,
  "total": 3,
  "results": [
    {
      "pdf_id": "1",
      "book_id": "f68dedc0-f8f5-4df7-a401-c77653a9b450",
      "upload_date": "2026-03-25T07:28:51.154458",
      "similarity": 1.0,
      "title": "1",
      "author": null,
      "isbn": null,
      "is_self": true
    }
  ]
}
```

## 四、POST /api/v2/books/hotlist

**行为简述**：筛选 `upload_date` 落在 start_date, end_date 内的上传；**threshold ≥ 1.0** 时每组即一个 `book_id`（仅按库内书目合并同一书的多 pdf）；**threshold < 1.0** 时用 `**fingerprint_mode` 对应索引**做向量相似合并（分数 ≥ threshold 的书目并入同一组）。组内 `pdf_ids` 为时间窗内出现过的上传 id，**组级 `upload_count` = `pdf_ids` 个数**（不是 `books` 表上的累计字段）。

**Request JSON**


| 字段               | 类型       | 必填  | 说明                                                      |
| ---------------- | -------- | --- | ------------------------------------------------------- |
| start_date       | datetime | 是   | ISO8601；与 `end_date` 闭区间过滤 `BookUpload.upload_date`     |
| end_date         | datetime | 是   | 同上                                                      |
| threshold        | float    | 否   | 默认 **1.0**；1.0 仅按 book 精确分组，<1.0 按指纹合并                  |
| top_n            | int      | 否   | 返回组数上限，默认 20，最大 100                                     |
| fingerprint_mode | string   | 否   | `merged` | `pooled`，默认 **merged**；仅 threshold<1.0 时用于合并 |


**Response JSON**


| 字段                                | 说明                                               |
| --------------------------------- | ------------------------------------------------ |
| message                           | HTTP 200 时为 `success`；4xx/5xx 见 FastAPI `detail` |
| time                              | 处理耗时（秒）                                          |
| start_date / end_date / threshold | 回显请求（时间多为 UTC 带 Z）                               |
| total_groups                      | 返回的组数（≤ top_n）                                   |
| results                           | 按组内 `upload_count`（pdf 条数）降序截取                   |


**results[] 每项**


| 字段                     | 说明                                    |
| ---------------------- | ------------------------------------- |
| book_ids               | 该组合并到的书目 id 列表                        |
| pdf_ids                | 时间窗内、属于这些书目的上传 pdf_id 列表              |
| pdf_urls               | 与 `pdf_ids` 等长，一一对应的 PDF URL 列表（可能为 null） |
| upload_count           | **本组 `pdf_ids` 的长度**（热门统计用）           |
| title / author / isbn  | 取组内 **representative_book_id** 对应书目展示 |
| representative_book_id | 组内用于展示元数据的 book_id（实现取每组第一个 book_id）  |


**示例（threshold<1.0 合并）**

```json
{
  "message": "success",
  "time": 0.042,
  "start_date": "2026-03-06T03:00:00.037000Z",
  "end_date": "2026-03-28T13:50:00.037000Z",
  "threshold": 0.8,
  "total_groups": 1,
  "results": [
    {
      "book_ids": ["f68dedc0-f8f5-4df7-a401-c77653a9b450"],
      "pdf_ids": ["1", "2", "3"],
      "upload_count": 3,
      "title": "1",
      "author": null,
      "isbn": null,
      "representative_book_id": "f68dedc0-f8f5-4df7-a401-c77653a9b450"
    }
  ]
}
```

## 五、错误码


| HTTP | 含义                                                           |
| ---- | ------------------------------------------------------------ |
| 200  | 成功；上传若 pdf_id 重复，body 内 `book_id` 为 `null` 且 `message` 为错误说明 |
| 400  | 参数或业务错误（如指纹向量不存在）                                            |
| 404  | search 时 pdf_id 未入库                                          |
| 500  | 下载失败、embedding 失败等                                           |


---

## 六、注意事项

- `VECTOR_STORE` 须为 `faiss`；Milvus 下书籍指纹接口不可用。
- 上传同步耗时大，建议客户端超时 ≥ 60 秒。
- `txt_url` 与 `pdf_url` 内容不一致时：哈希以 PDF 为准，指纹以正文为准。
