# 接口文档（v2 书籍）

**用途**：描述当前线上对外 HTTP 契约（上传 / 相似查询 / 热门）。实现细节见 `项目结构说明.md`，本地跑通见 `DEMO_本地运行指南.md`。

---

二、接口文档

接口1 — 上传书籍

POST /api/v2/books/upload

说明：临时下载 PDF 计算文件哈希（SHA256）并读取元数据（ISBN、标题等）后删除临时文件，不持久化 PDF。全文从 txt_url 下载，用于指纹向量与本地落盘（`data/texts/{book_id}.txt`）。指纹按伪页采样（前 3 + 中 3 + 后 2 + 标题；伪页数不足 8 则用全文），同时计算 merged、pooled 两套向量并写入两套 FAISS。同一本书若命中 content_hash / isbn / 向量（见 fingerprint_mode）则不新建书目，只记一条上传；新书会写入双指纹索引。

Request Body：

```json
{
  "pdf_id": "book_20260305_001",
  "pdf_url": "https://example.com/files/book.pdf",
  "txt_url": "https://example.com/files/book.txt",
  "fingerprint_mode": "merged"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pdf_id | string | ✅ | 业务唯一 ID；已存在时返回 HTTP 200 且 body 中 `status` 为 `error` |
| pdf_url | string | ✅ | PDF 可下载 URL，仅用于哈希与元数据 |
| txt_url | string | ✅ | 纯文本 URL（建议 UTF-8），指纹与落盘均基于此正文 |
| fingerprint_mode | string | ❌ | `merged`（拼接片段后一次 embedding）或 `pooled`（分段 embedding 再加权平均）；默认 `merged`。去重时向量匹配走该模式对应索引 |

Response：

```json
{
  "status": "success",
  "book_id": "3f8a2c1d-...",
  "pdf_id": "book_20260305_001",
  "is_duplicate": false,
  "match_reason": null,
  "upload_count": 1
}
```

| 字段 | 说明 |
|------|------|
| is_duplicate | true 表示判为同一本书，未再写指纹索引 |
| match_reason | `content_hash` / `isbn` / `vector`（向量分支使用请求中的 fingerprint_mode 对应索引） |
| upload_count | 该书累计上传次数 |

接口2 — 相似书籍查询

POST /api/v2/books/search

说明：用该书已入库的指纹向量，在选定模式的 FAISS 索引中检索，不再对正文实时 embedding。

Request Body：

```json
{
  "pdf_id": "book_20260305_001",
  "threshold": 0.8,
  "fingerprint_mode": "merged"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pdf_id | string | ✅ | 已上传过的 pdf_id |
| threshold | float | ❌ | 相似度下限，默认 0.8 |
| fingerprint_mode | string | ❌ | `merged` / `pooled`，默认 `merged`，须与入库时写入的索引一致才有意义 |

Response 结构同前；`similarity` 为指纹向量内积（归一化后等价余弦相关）。

接口3 — 热门书籍

POST /api/v2/books/hotlist

说明：`threshold=1.0` 时按 book_id 精确分组；`threshold<1.0` 时用指纹向量合并相似书再统计。合并所用索引由 `fingerprint_mode` 决定。

Request Body：

```json
{
  "start_date": "2026-03-01T00:00:00",
  "end_date": "2026-03-05T23:59:59",
  "threshold": 1.0,
  "top_n": 20,
  "fingerprint_mode": "merged"
}
```

| 字段 | 默认值 | 说明 |
|------|--------|------|
| threshold | 1.0 | 1.0 仅按同一 book；小于 1.0 时按指纹相似度合并 |
| top_n | 20 | 返回前 N 组 |
| fingerprint_mode | merged | `merged` / `pooled` |

Response 结构同前。

三、错误码

| HTTP | 含义 |
|------|------|
| 200 | 成功（上传若 pdf_id 重复，body 内 `status` 可能为 `error`） |
| 400 | 参数或业务错误（如该书指纹向量不存在） |
| 404 | pdf_id 在库中不存在 |
| 500 | 内部错误（URL 下载失败、embedding 失败等） |

---

四、注意事项

- `VECTOR_STORE` 须为 `faiss`：指纹双索引仅实现于 FAISS；生产若配 Milvus，书籍指纹相关接口不可用。
- 上传为同步阻塞：下载 + 多段 embedding + 写盘，建议客户端超时 ≥ 60 秒。
- `txt_url` 正文质量直接影响指纹效果；`pdf_url` 与正文不一致时，以正文为准做指纹，哈希仍以 PDF 字节为准。
- search / hotlist 使用已存向量检索，耗时主要取决于 FAISS 规模，一般秒级内。
